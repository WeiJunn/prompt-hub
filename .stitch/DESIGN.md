# Prompt 浏览器设计规范

## 定位
- 这是一个 Prompt 浏览器，不是内容管理后台。
- 核心任务只有 3 个：找得到、看得懂、复制走。
- 页面优先级：分类清晰、搜索快速、详情干净。

## 品牌与氛围
- 气质：工具型内容站、清爽、专业、可信赖。
- 页面密度：中高信息密度，但通过卡片、边框和留白维持秩序感。
- 视觉关键词：editorial utility, content-first, soft cards, crisp search surfaces.

## 设计系统
- 平台：Web，桌面优先并兼容移动端。
- 背景色：`#f6f8f7`
- 主色：`#12344d`，用于主按钮、标题强调、导航高亮。
- 辅色：`#0f766e`，用于分类标识、筛选态、次级强调。
- 强调色：`#f59e0b`，用于复制、热度、结果高亮。
- 文本主色：`#1f2933`
- 次级文本：`#5b6875`
- 边框：`#d7dde4`

## 字体
- 英文：Inter
- 中文：Noto Sans SC
- 等宽：JetBrains Mono

## 组件原则
- 卡片圆角大于常规后台页面，采用柔和阴影与细边框。
- 搜索框始终可快速触达，移动端顶部保持 sticky。
- 标签、筛选和 breadcrumb 都应保持紧凑，不夺走正文注意力。
- 详情页默认突出复制按钮和正文阅读区。

## 页面结构
1. 首页：Hero、分类卡片、最近更新、热门 Prompt。
2. 分类页：面包屑、简介、搜索、标签筛选、Prompt 列表。
3. 详情页：标题、简介、复制、纯文本/渲染切换、侧边信息、相关推荐。
4. 搜索页：即时过滤、命中高亮、分类与标签联动。
